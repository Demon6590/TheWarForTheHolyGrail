ЗАДАНИЕ.
Игра "Герои" (текстовый hack-and-slash).
Необходимо реализовать несколько разных героев, которые могут атаковать друг друга.
Необходимо реализовать возможность сохранения и восстановления состояния героя.
Необходимо реализовать текстовый интерфейс для игры.
Над балансом игры можно не думать.

Также необходимо применять паттерны и принципы SOLID, желательно сделать слоёную архитектуру приложения, желательно заложить возможность дальнейшей доработки приложения.

Для начала нужно сделать диаграммы для понимания что и как будет происходить в игре

Диаграмма действий

```mermaid
flowchart TD
    A[Запуск игры] --> B[Инициализация GameController]
    B --> C[Инициализация TextUI]
    C --> D[Отображение главного меню]
    D --> E{Выбор действия}
    
    E -->|Создать героя| F[Вызов CreateHero в GameController]
    F --> G[Выбор типа героя Warrior/Mage/Archer]
    G --> H[Использование Builder]
    H --> I[Настройка параметров героя]
    I --> J[Build героя]
    J --> K[Добавление в список]
    K --> D
    
    E -->|Атаковать| L[Вызов Attack]
    L --> M[Выбор атакующего]
    M --> N[Выбор цели]
    N --> O[Attack атакующего]
    O --> P[Расчёт урона]
    P --> D
    
    E -->|Сохранить| Q[Вызов SaveGame]
    Q --> R[SaveLoadService]
    R --> S[Json сериализация]
    S --> T[Сохранение в файл]
    T --> D
    
    E -->|Загрузить| U[Вызов LoadGame]
    U --> V[Загрузка из файла]
    V --> W[Json десериализация]
    W --> X[Восстановление героев]
    X --> D
    
    E -->|Выход| Y[Завершение]
```

Диаграмма состояния:

```mermaid
stateDiagram-v2
    [*] --> Alive
    
    state Alive {
        [*] --> Idle
        Idle --> Attacking : Attack()
        Attacking --> Idle
        
        Idle --> TakingDamage : ReceiveDamage(dmg)
        TakingDamage --> Idle : HP > 0
        TakingDamage --> Dead : HP <= 0
    }
    
    state Dead {
        [*] --> Finished  direction LR
        Finished --> [*]
    }
    
    Alive --> Dead
```
